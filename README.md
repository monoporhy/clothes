# 服カタログ

手持ちの服を管理・公開する個人リファレンスサイト。

**公開URL**: https://monoporhy.github.io/clothes/

- [服カタログ](https://monoporhy.github.io/clothes/) — 全アイテム一覧
- [コーデの提案](https://monoporhy.github.io/clothes/coordinate.html) — ランダムコーデ提案

**主な用途**: 買い物時に「似たものを持っていないか・サイズは合うか」を確認する。

---

## 構成

```
clothes/
├── index.html         # 服カタログ（全アイテム一覧・カテゴリタブ・ソート）
├── coordinate.html    # コーデ提案（トップス・ボトムス・アウターをランダムピック）
├── clothes.json       # 服データ（source of truth）
└── images/            # 商品画像（{id}.jpg / .png / .webp）
```

## ページ仕様

- `clothes.json` をランタイムに fetch して描画（ビルドステップなし）
- GitHub Pages で自動公開（main push → 即反映）
- ダーク系UIで統一（`#0b0c0e` ベース、スチールブルーアクセント）

### index.html — 服カタログ

#### カテゴリタブ

ALL / トップス / ボトムス / アウター / その他

#### テーブル

| 列 | 内容 |
|---|---|
| Cat | カテゴリ |
| Brand | ブランド名 |
| Item | 商品名（長い場合は省略、クリックで展開） |
| Color | カラー |
| 着丈 / 肩幅 / 身幅 / 袖丈 | 寸法（cm）、600px以下では非表示 |

- 各列ヘッダーをクリックで昇順/降順ソート
- レスポンシブ対応（スマホでは寸法列を非表示・横スクロール）

### coordinate.html — コーデ提案

- トップス・ボトムス・アウターをランダムで1点ずつピックして3点コーデを提案
- "Shuffle" ボタンで再生成
- 画像なし時は "NO IMAGE" プレースホルダー表示
- レスポンシブ対応（600px以下でカードが縦並び）

## アイテム追加方法

Claude Code に商品URLを渡すだけで2ステップで追加できる。

1. **URLを貼る** → ブランド・商品名・カテゴリを取得して確認、og:image を `images/{id}.jpg` にダウンロード
2. **サイズとカラーを伝える** → 寸法を取得して `clothes.json` に追記、`images/` と合わせて push

## 寸法の一括更新（CSV）

スプレッドシートからCSVを貼ると、**名称で突合**して寸法を更新できる。

CSVの列順: `カテゴリ, ブランド, 名称, カラー, サイズ, 着丈, 肩幅, 身幅, 袖丈, ...`

- 名称が完全一致しない場合は部分一致でフォールバック
- 同名で複数カラーがある場合はカラーでさらに絞り込み
- 寸法列が空の行はスキップ

## データ形式

```json
{
  "id": "brand-color-size-yyyy-mm",
  "category": "トップス",
  "brand": "ブランド名",
  "name": "商品名",
  "color": "カラー",
  "size": "サイズ",
  "dimensions": { "着丈": 69, "肩幅": 44, "身幅": 52, "袖丈": 20 },
  "url": "https://...",
  "image": "images/{id}.jpg",
  "purchased_at": "2026-03"
}
```

カテゴリ: `トップス` / `ボトムス` / `アウター` / `シューズ` / `バッグ` / `アクセサリー` / `その他`

寸法キー:
- トップス / アウター: `着丈`, `肩幅`, `身幅`, `袖丈`
- ボトムス: `ウエスト`, `股上`, `渡り`
- シューズ / バッグ / アクセサリー: `{}` (空)
